# 2026-04-01 - Reparación de Mission Control

## Problema reportado
Mission Control se quedaba en "loading default" y no cargaba.

## Diagnóstico
- Contenedor en estado `unhealthy`
- Logs mostraban error `SQLITE_CANTOPEN: unable to open database file`
- El problema era que el contenedor corría como usuario `node` (UID 1000) pero los volúmenes montados desde el host pertenecían a `ubuntu` (UID 1001)

## Solución
1. Añadido `user: "1001:1001"` al `docker-compose.yml`
2. Reconstruido y reiniciado el contenedor

## Verificación post-reparación
- ✅ Contenedor saludable (`health: starting` → `healthy`)
- ✅ Base de datos SQLite creada correctamente
- ✅ API de health responde: `{"status":"ok","db":{"writable":true}}`
- ✅ API de events funciona (retorna `[]` vacío pero sin errores)
- ✅ Dashboard accesible via Tailscale

## Archivos modificados
- `~/mission-control/docker-compose.yml` - Añadido mapeo de usuario UID 1001:1001
